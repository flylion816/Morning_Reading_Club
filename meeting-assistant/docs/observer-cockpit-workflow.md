# 晨读观察台工作流

## Product Positioning

晨读观察台不是 AI 聊天框，而是观察者现场工作台：

```text
采集多位发言 -> 单人抓本质 -> 匹配本期主题 -> 生成现场回应 -> 合并全场洞察
```

第一版不依赖实时语音转写。输入可以来自腾讯会议转写、截图 OCR、聊天文本、手动速记或后续 ASR。

## Speaker Card

每位发言人有独立状态和独立输出。

```ts
type SpeakerCard = {
  name: string
  status: 'empty' | 'captured' | 'analyzed' | 'response_ready' | 'responded'
  snippets: ContentSnippet[]
  insight?: SpeakerInsight
  actualResponse?: string
}
```

## Content Snippet

同一个人可以有多段输入，不强制一次粘完整。

```ts
type ContentSnippet = {
  sourceType: 'manual_paste' | 'clipboard' | 'ocr_text' | 'meeting_transcript' | 'future_asr'
  text: string
  createdAt: string
}
```

## Single Speaker Output

单人分析不是长文，而是现场可用的结构化结果：

- 最值得抓的点
- 背后的本质
- 对应七个习惯主题
- 卡点分类
- 他需要被看见的地方
- 适合关联观察者的哪段经历
- 20 秒回应
- 1 分钟回应
- 有力量的追问
- 金句
- 不要说的话

## Whole Session Output

全场合并在所有人分析后执行，但也允许中途生成草稿。

- 今天共同主题
- 每个人的生命课题
- 每个人和本期主题的关系
- 观察者如何统一回应
- 最终 1000-1500 字总结
- 5 句金句
- 最后一句收束

## Operating Principles

- 现场操作必须少：粘贴、分析、复制回应都要一键完成。
- 不让 AI 自由聊天：所有输出必须进入固定字段。
- 先服务单人回应，再服务全场总结。
- 每次输出都必须提醒“不要说的话”。
- 音频、OCR、腾讯会议 webhook 都只是输入源，不能改变主工作流。
