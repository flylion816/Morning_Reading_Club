## 🔐 关键特性验证

### ✅ 财务安全（最重要）

**并发支付防护** (3 个测试用例)
```
实现方式：乐观锁 + 状态检查
测试场景：
  1. 两个并发请求支付同一报名 → 第二个拒绝
  2. 支付完成后再次支付 → 返回 400
  3. 支付中状态再次支付 → 返回已存在
```

**金额验证** (9 个测试用例)
```
拒绝条件：
  - 零金额 (0)
  - 负金额 (-100)
  - 小数金额 (99.99)
  - 超大金额 (>1000000)

接受条件：
  - 正整数 (1 ~ 1000000)
  - 最小单位 1 分
  - 合理上限 10000 元
```

**事务一致性** (2 个测试用例)
```
支付成功：
  Payment.status = 'completed'
  Enrollment.paymentStatus = 'paid'
  Enrollment.paidAt = now()

支付失败：
  Payment.status = 'failed'
  Enrollment.paymentStatus = 'pending' (不变)
```
