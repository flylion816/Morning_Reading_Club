## 🏗️ 架构设计

### Method C: Hybrid Reuse 架构

```
┌──────────────────────────────────────────────────┐
│         主部署脚本 (6 个，线性执行)               │
│  1️⃣  2️⃣  3️⃣  4️⃣  5️⃣  6️⃣               │
└────────────┬──────────────────────────────────────┘
             │
     ┌───────┴───────┬──────────────┬──────────────┐
     │               │              │              │
┌────▼────┐  ┌──────▼──────┐  ┌────▼──────┐  ┌───▼───┐
│ Deploy  │  │  Database   │  │ Infra     │  │ Utils │
│Functions│  │  Functions  │  │ Functions │  │ (Base)│
│(14 fn)  │  │  (17 fn)    │  │ (15 fn)   │  │(19 fn)│
└────┬────┘  └─────┬───────┘  └────┬──────┘  └───┬───┘
     │            │               │            │
     └────────────┴───────────────┴────────────┘
         所有库都依赖于 utils.sh（基础）
```

### 优点

✅ **代码复用**: 每个函数只写一次，65 个函数被 6 个脚本复用
✅ **脚本独立**: 每个脚本 70-170 行，职责单一，易于理解
✅ **易于扩展**: 添加新脚本只需 source 相应的库
✅ **易于维护**: 修改函数只需改库，无需修改多个脚本
✅ **易于测试**: 每个函数都可以单独测试
✅ **性能稳定**: 所有脚本调用相同的库函数，行为一致

## 📈 完整部署流程

### 全新服务器部署 (15-25 分钟)

```
START
  ↓
[1-initial-setup.sh]
  • 验证 sudo 权限
  • 检查系统（架构、磁盘）
  • 创建应用目录
  ↓ (2-3 min)
[2-install-dependencies.sh]
  • 安装 Node.js、npm
  • 安装 Docker、Docker Compose
  • 安装 PM2、Nginx、Certbot
  ↓ (5-8 min)
[3-setup-infrastructure.sh]
  • 克隆 Git 仓库
  • 创建必要目录
  • 设置权限
  ↓ (1-2 min)
[4-init-database.sh]
  • 启动 Docker 容器
  • 初始化数据库
  • 运行初始化脚本
  ↓ (3-5 min)
[5-setup-nginx.sh]
  • 配置 Nginx
  • 申请 SSL 证书
  • 启动 Nginx
  ↓ (2-3 min)
[6-deploy-app.sh]
  • 安装后端依赖
  • 启动 PM2 应用
  • 部署管理后台
  ↓ (2-3 min)
[verify-deployment.sh]
  • 验证所有组件
  ✓ DEPLOYMENT SUCCESSFUL
```
