# 账号切换改造 TODO

当前决定先不启动“切换账号 / 手机号登录”改造，等短信能力或更明确的账号策略后再继续。

## 已确认方向

- 平台账号与登录方式分离
- 支持微信登录与手机号登录并存
- 用户资料、报名、支付、打卡、通知都归属平台账号，不归属某个单独微信身份
- “切换账号”最终应支持重新选择登录方式，而不只是退出后回到同一个微信账号

## 暂缓原因

- 代码库当前没有短信验证码能力
- 没有发送验证码、验证码校验、过期时间、限流、风控等配套后端
- 直接做“手机号验证码登录”会停留在 UI 层，无法完整落地

## 后续分阶段建议

- 去掉“我的”页头像旁边重复的“编辑”入口
- 保留“编辑资料”作为唯一资料编辑入口
- 维持现有微信登录 + 手机号绑定能力

### 阶段 2：账号体系改造

- 设计平台账号与身份凭证模型
- 明确微信身份、手机号与平台账号的绑定关系
- 处理绑定冲突和账号合并策略
- 输出数据迁移方案

### 阶段 3：短信验证码登录

- 接入短信服务商
- 增加发送验证码接口
- 增加验证码校验、过期、限流与审计
- 登录页增加手机号验证码登录入口
- “切换账号”真正支持切到手机号账号

## 需要后续确认的问题

- 是否要支持账号合并，还是先禁止自动合并
