## 一、背景与目标

### 背景

晨读营书友除了线上学习，也会自发组织线下活动：一起做饭、读书会、喝茶碰头、散步等。这些线下相遇目前没有统一的记录和分享渠道，散落在个人朋友圈或微信群，线上书友无法感知，参与者也缺乏一个有仪式感的留存方式。

### 目标

在晨读营小程序内开辟「在场」频道，让书友能够：
1. 方便地记录线下聚会的照片/视频
2. 标记参与者（含非注册用户）
3. 围绕这些内容互动（评论、共鸣）
4. 让线上书友感受到线下社群的温度

### 不做什么

- 不做直播、实时位置共享
- 不做复杂的活动报名/组织功能（那是另一个产品）
- 不做陌生人社交，仅限晨读营社群内部

---

## 二、用户与场景

### 用户群体

| 角色 | 描述 |
|------|------|
| 发布者 | 参与线下活动的注册书友，负责拍照发布 |
| 在场者 | 同场参与的书友，可被 @ 或自行认领"我也在场" |
| 旁观者 | 未参与但在线上浏览的书友，通过内容感受线下氛围 |
| 非注册参与者 | 参加了活动但未注册小程序的书友，由发布者手动填写名字 |

### 核心场景

1. **聚会现场发布**：活动进行中或结束后，掏出手机，30 秒内完成发布
2. **事后补记**：活动结束几天后，整理照片补发
3. **线上浏览**：未参与的书友刷「在场」频道，感受线下社群活力
4. **找到同城书友**：通过地图视图发现同城有人在聚，产生连接

## 三、功能详述

### 3.1 内容单元：「印记」

每条发布内容称为**「印记」**（Imprint），是「在场」的基本单元。

#### 印记字段

| 字段 | 必填 | 说明 |
|------|------|------|
| 图片/视频 | 是 | 最多 9 张图片，或 1 段视频（≤60s）；图片和视频不能混选 |
| 标题 | 是 | 最多 30 字，如"We来坊，下午茶" |
| 活动类型 | 是 | 单选，见活动类型列表 |
| 文字描述 | 否 | 最多 200 字 |
| 地点 | 否 | 手动输入地点名称，最好支持获取地理位置，最多 20 字 |
| 在场人 | 否 | @注册书友 或 手动输入名字（非注册用户） |
| 所属营期 | 否 | 关联到某个营期，可不填 |
| 发布时间 | 自动 | 默认当前时间，允许修改（支持补记历史活动） |

#### 活动类型（标签）

活动类型当前已改为后台动态配置，以下为默认兜底类型；管理后台可按租户维护 key、名称、emoji、排序和启用状态。

| 类型 | 图标 | 颜色 |
|------|------|------|
| 读书会 | 📚 | 墨绿 #2D5A3D |
| 一起做饭 | 🍳 | 橙色 #C4873A |
| 喝茶聊天 | ☕ | 棕色 #8B6355 |
| 散步 | 🚶 | 蓝色 #4A7FA5 |
| 创作 | 🎨 | 紫色 #7B5EA7 |
| 其他 | ✨ | 灰色 #888888 |

当前代码已补充运营类型，例如「料理人生活」。

### 3.2 封面拼图规则

多张图片自动拼合为封面，不做轮播：

| 图片数量 | 拼图样式 |
|----------|----------|
| 1 张 | 全幅单图 |
| 2 张 | 左右各半 |
| 3 张 | 上方一大图，下方两小图 |
| 4 张 | 田字格 |
| 5-9 张 | 第一张大图占左侧，右侧 2×2 网格展示后4张，其余显示 "+N" |

### 3.3 在场人系统

**发布时添加**：
- @注册书友：搜索昵称，选择后关联用户账号
- 手动输入：填写名字字符串，不关联账号（用于非注册参与者）

**事后认领**：
- 注册书友看到印记后，可点击「我也在场」，头像加入在场人列表
- 发布者可以移除错误认领

**在场人展示**：
- 印记卡片底部显示头像墙，最多展示 8 个头像
- 超出显示 "+N人在场"
- 点击头像墙展开完整在场人列表

### 3.4 互动：「共鸣」

用三种有温度的反应替代普通点赞：

| 反应 | 含义 | 场景 |
|------|------|------|
| 🌱 共鸣 | 我感受到了 | 默认，最常用 |
| 🔥 燃 | 好燃！受到激励 | 活动很精彩 |
| 🤗 想去 | 下次带我 | 表达参与意愿 |

每人每条印记只能选一种反应，可切换。

### 3.5 评论

- 支持文字评论
- 支持回复某条评论（一级嵌套，不做多级）
- 评论按时间正序排列
- 发布者可删除评论

### 3.6 分享功能

当前已实现微信分享给好友/朋友圈，列表页和详情页优先使用首张图片作为分享图。以下「在场卡片」长图仍是后续目标：

```
┌─────────────────────────┐
│  2026.05.22  We来坊·北京 │
│                         │
│  [封面拼图]              │
│                         │
│  下午茶                  │
│  可心 · 敏慧 · +3人在场  │
│                         │
│  我在场，我见证          │
│  ─────────────────────  │
│  [晨读营小程序码]        │
└─────────────────────────┘
```

- 可保存到相册
- 可分享到微信好友/朋友圈
- 带小程序码，扫码可跳转到该印记详情

### 3.7 浏览与筛选

#### 默认视图：时间流

- 当前代码按更新时间 `updatedAt` 倒序，编辑过的印记会回到前面
- 大图卡片流，每屏约 1.5 张卡片，引导向下滚动
- 年月分隔线（如"2026年5月"）

#### 地图视图（未实现）

- 所有有地点信息的印记，按地点聚合显示在地图上
- 地图圆点：同一地点多条印记聚合，显示数量
- 点击圆点：展开该地点的印记列表
- 无地点信息的印记不在地图上显示

#### 筛选条件

| 维度 | 选项 |
|------|------|
| 活动类型 | 全部 / 读书会 / 做饭 / 喝茶 / 散步 / 创作 / 其他 |
| 时间 | 全部 / 最近一月 / 最近三月 / 自定义 |
| 营期 | 全部 / 第N期（下拉选择） |

筛选条件以横向滚动 chip 形式展示，不做弹窗。

### 3.8 已上线增强

- **视频支持已上线**：发布页支持选择 1 个视频，视频最长 60 秒，大小限制 50MB；图片和视频不能混选。列表和发布页使用 video 封面/播放标识，点击后通过 `wx.previewMedia` 全屏预览。
- **编辑/删除已上线**：作者可在详情页编辑或删除自己的印记；编辑页复用发布页，可修改标题、活动类型、地点、描述、媒体和在场人。
- **分享已上线**：列表页和详情页支持分享给好友/朋友圈，优先预下载首张图片作为分享图；视频印记没有图片时使用默认分享信息。
- **通知已上线**：共鸣、评论、注册用户被添加为在场人时，会触发站内通知和订阅消息。
- **行为埋点已上线**：列表浏览、详情浏览、发布、共鸣、评论等操作写入用户行为记录，用于运营统计。
- **首页露出已上线**：首页会展示近期在场印记卡片，并提供进入在场频道的入口。

## 五、视觉风格

### 色彩系统

| 用途 | 色值 |
|------|------|
| 页面背景 | #F5F0E8（暖米色） |
| 主色（按钮、强调） | #2D5A3D（深墨绿） |
| 强调色（数字、徽章） | #C4873A（琥珀橙） |
| 正文文字 | #2C2C2C |
| 次要文字 | #888888 |
| 卡片背景 | #FFFFFF |
| 卡片阴影 | rgba(0,0,0,0.06) |

### 设计原则

- **手账感**：整体像一本线下聚会的记录本，不像社交APP
- **大图优先**：图片是主角，文字是注脚
- **克制留白**：卡片间距宽松，不拥挤
- **圆角柔和**：卡片圆角 12px，图片圆角 8px

### Tab 图标

「在场」Tab 图标：两个人形轮廓靠近，简洁线条风格。

## 七、权限与边界

| 操作 | 权限 |
|------|------|
| 浏览印记列表 | 登录用户 + 已付费/免费授权报名 |
| 查看印记详情 | 可选登录；分享落地可查看，互动需登录 |
| 发布印记 | 登录用户 + 已付费/免费授权报名 |
| 编辑自己的印记 | 仅发布者 |
| 删除自己的印记 | 仅发布者 |
| 点共鸣 | 登录用户 |
| 发评论 | 登录用户 |
| 「我也在场」 | 登录用户 |
| 删除评论 | 发布者可删除任意评论；用户可删除自己的评论 |
| 管理后台审核/删除 | 管理员 |

说明：当前后端用 `requirePaidEnrollment` 校验 `paymentStatus in ['paid','free']` 且 `status in ['active','completed']` 的报名记录；小程序端也会在进入列表和发布页前做同样守卫。

## 八、数据模型（概要）

### Imprint（印记）

```
_id, tenantId
title           // 标题
description     // 描述
activityType    // 活动类型枚举
location        // 地点名称（字符串）
coordinates     // [lng, lat]（可选，用于地图）
mediaList       // [{type: image/video, url, thumbUrl}]
attendees       // [{userId, name, isRegistered}]  在场人
periodId        // 关联营期（可选）
authorId        // 发布者
happenedAt      // 活动发生时间（允许历史时间）
createdAt, updatedAt
reactionCounts  // {gonming: N, ran: N, xiangqu: N}
commentCount
```

### ImprintReaction（共鸣）

```
_id, tenantId
imprintId, userId, type  // type: gonming/ran/xiangqu
createdAt
```

### ImprintComment（评论）

```
_id, tenantId
imprintId, authorId
content
parentId        // 回复某条评论（可选）
createdAt
```

## 九、接口概要

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/v1/imprints | 获取印记列表（支持筛选分页） |
