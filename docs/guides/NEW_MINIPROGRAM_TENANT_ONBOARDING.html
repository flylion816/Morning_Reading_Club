晨读营多租户小程序开通流程

新开一个晨读营小程序操作指南

为新的品牌或机构开通独立微信小程序前端，同时继续使用晨读营统一中台和后端能力。页面按业务准备、中台配置、技术上传、验收发布的实际顺序组织。

适用对象业务方、运营同学、晨读营中台管理员、技术发布人员
适用场景新增一个独立 appId 的品牌/机构小程序前端
核心原则同一套代码按租户配置生成并上传，不复制新仓库

最终发布链路

业务方准备微信小程序资料
晨读营中台创建租户并录入配置
技术侧放置租户素材
本地一键同步配置并上传开发版
微信后台提交审核
审核通过后发布

源文件：docs/guides/NEW_MINIPROGRAM_TENANT_ONBOARDING.md

一、整体说明

新开一个前端晨读营，本质上是新增一个“租户小程序壳”：

  • 业务方拥有一个独立微信小程序 appId。
  • 小程序品牌名、颜色、Logo、Tab 图标、订阅消息模板、支付配置可以独立配置。
  • 后端仍使用晨读营统一中台，通过租户配置隔离用户、报名、打卡、支付、内容等数据。
  • 小程序代码不复制新仓库，由同一套代码按租户配置生成并上传。

最终发布链路：

text业务方准备微信小程序资料
→ 晨读营中台创建租户并录入配置
→ 技术侧放置租户素材
→ 本地一键同步配置并上传小程序开发版
→ 微信后台提交审核
→ 审核通过后发布

二、业务方需要准备的资料

1. 微信小程序主体

业务方需要先在微信公众平台完成：

  • 注册新的微信小程序。
  • 完成主体认证。
  • 完成 ICP 备案。
  • 配置服务类目，类目需与实际业务一致。
  • 配置用户隐私保护指引。

需要提供给晨读营中台的信息：

项目说明
小程序名称例如：凡人共读、某某晨读营
appId形如 wx2b9a3c1d5e4195f8
AppSecret微信登录用，敏感信息，单独安全发送
小程序管理员微信用于体验、审核、问题确认

2. 微信支付资料

如果新小程序需要收费报名，业务方需要准备：

项目说明
微信支付商户号 mchId绑定到该小程序
支付 API Key / API v3 Key按当前支付配置要求提供
商户证书如支付流程需要证书，由技术侧确认格式
支付回调授权确认商户平台允许当前后端回调域名

如果暂时不启用支付，可以先不配置支付信息。

3. 品牌素材

业务方需要提供：

素材要求
Logo建议 PNG，正方形，清晰透明背景优先
小程序头像按微信公众平台要求上传
分享封面建议 JPG/PNG，适合微信分享卡片
主色调十六进制颜色，例如 #4a90e2
Tab 图标首页、晨读营、我的，每个都要普通态和选中态

Tab 图标共 6 个：

texttab-home.png
tab-home-active.png
tab-book.png
tab-book-active.png
tab-my.png
tab-my-active.png

4. 订阅消息模板

业务方需要在微信公众平台申请订阅消息模板，并提供模板 ID。

常用场景包括：

  • 报名结果通知
  • 支付结果通知
  • 评论通知
  • 点赞通知
  • 次日学习提醒
  • 活动提醒

中台中以 JSON 方式保存，例如：

json{
  "enrollment_result": "模板ID",
  "payment_result": "模板ID",
  "next_day_study_reminder": "模板ID"
}

三、微信公众平台需要配置

业务方或小程序管理员需要在微信公众平台配置：

1. 服务器域名

需要把晨读营线上域名加入合法域名：

texthttps://wx.shubai01.com

常见配置项：

  • request 合法域名
  • uploadFile 合法域名
  • downloadFile 合法域名
  • socket 合法域名，如果该小程序启用实时能力

2. 插件授权

如果小程序使用了微信插件，例如语音、识别或其它能力，需要在新 appId 下重新授权。插件授权是按 appId 生效的，不会自动继承旧小程序。

3. 代码上传密钥

需要在微信公众平台生成“代码上传密钥”，交给技术侧放到本地：

textminiprogram/keys/<slug>.key.pem

示例：

textminiprogram/keys/fanren.key.pem
miniprogram/keys/chaoren.key.pem

密钥文件不能提交到 Git。

四、晨读营中台需要做的事

1. 新建租户

登录晨读营管理后台：

texthttps://wx.shubai01.com/admin/tenants

点击“新建租户”，填写：

字段示例说明
Slugfanren租户短标识，只能小写字母/数字/下划线/连字符，创建后不要修改
名称凡人共读后台显示名称
AppId 列表wx...新小程序 appId
登录 AppIdwx...通常与 AppId 列表一致
登录 SecretAppSecret微信登录用
支付 AppIdwx...启用支付时填写
商户号mchId启用支付时填写
支付密钥API Key启用支付时填写
云环境 ID可选如启用云开发则填写
品牌名凡人共读小程序内展示
主色调#4a90e2小程序主题色
Logo URL可留空留空默认使用本地租户素材
分享图 URL可留空留空使用默认分享图
主体名称可选协议页主体名称
联系邮箱可选协议页或客服联系邮箱
模板 JSON订阅消息模板 ID按场景键配置

保存后，配置已经进入线上后端数据库。

2. 创建租户管理员

在账号管理中创建该租户的管理员账号：

  • 角色选择 tenant_admin。
  • 绑定到新租户。
  • 交付给对应业务方或运营负责人。

3. 配置业务内容

新租户创建后，中台需要继续配置：

  • 期次
  • 课程内容
  • 报名信息
  • 支付价格
  • 优惠券或活动
  • 打卡配置
  • 活动配置
  • 订阅消息触达策略

这些配置属于业务运营配置，不需要重新发布小程序代码，除非涉及品牌壳、appId、Tab 图标等构建期配置。

五、技术侧需要做的事

1. 放置租户素材

在代码仓库中新增目录：

textminiprogram/assets/tenants/<slug>/

至少放入：

textlogo.png
share-cover.jpg
tab-home.png
tab-home-active.png
tab-book.png
tab-book-active.png
tab-my.png
tab-my-active.png

2. 放置上传密钥

把微信公众平台生成的代码上传密钥放到：

textminiprogram/keys/<slug>.key.pem

例如：

textminiprogram/keys/fanren.key.pem

3. 一键同步并上传

推荐使用交互脚本：

bash./upload-miniapp-tenant.command <slug>

示例：

bash./upload-miniapp-tenant.command fanren

脚本会提示输入：

  • 版本号，可留空
  • 上传说明，可留空
  • Admin API 地址，默认 https://wx.shubai01.com/api/v1
  • 管理后台邮箱
  • 管理后台密码

脚本会自动完成：

text读取线上中台租户配置
→ 生成 miniprogram/config/tenants/<slug>.js
→ 切换小程序 appId、主题色、Tab 图标
→ 通过 miniprogram-ci 上传开发版

也可以使用命令行：

bashnpm run tenant:push -- <slug>

如果不用交互脚本，需要提供后台登录信息：

bashnpm run tenant:push -- <slug> --email <后台邮箱> --password <后台密码>

不建议长期使用带密码的命令行，因为密码可能留在 shell 历史中。

4. 上传后检查

上传完成后，在微信公众平台检查：

  • 开发版 appId 是否正确。
  • 版本号和上传说明是否正确。
  • 小程序名称、Logo、主色调是否正确。
  • Tab 图标是否正确。
  • 登录是否进入对应租户。
  • 报名、支付、订阅消息是否使用对应租户配置。

5. 提交审核和发布

tenant:push 只负责上传开发版，后续仍需人工在微信公众平台完成：

  1. 设为体验版并测试。
  2. 提交微信审核。
  3. 审核通过后发布。

六、验收清单

业务方验收：

  • 小程序名称、头像、简介正确。
  • 首页品牌展示正确。
  • 登录成功后进入对应新租户。
  • 课程、期次、报名内容正确。
  • 支付流程可用，如启用支付。
  • 订阅消息能正常触发，如已配置模板。
  • 分享卡片标题和图片正确。

中台验收：

  • 租户 slug 唯一且不可修改。
  • wxAppIds 和登录 AppId 与微信公众平台一致。
  • AppSecret 已保存。
  • 支付配置已保存，或明确暂不启用支付。
  • 订阅模板 JSON 格式正确。
  • 已创建租户管理员。
  • 已配置首期业务内容。

技术验收：

  • miniprogram/assets/tenants/<slug>/ 素材齐全。
  • miniprogram/keys/<slug>.key.pem 已放置且未提交。
  • ./upload-miniapp-tenant.command <slug> 上传成功。
  • 微信后台开发版 appId 正确。
  • 开发者工具或体验版测试通过。

七、常见问题

1. 后台保存后为什么还要本地运行脚本？

因为微信小程序的 appId、Tab 图标、主题等属于构建期配置。后台保存的是线上租户数据，本地脚本负责在上传前读取这些数据并生成小程序构建文件。

2. 本地是否需要连接服务器 MongoDB？

不需要。正常流程通过线上后端 Admin API 读取租户配置。

3. 为什么不能只在中台点一下就发布？

微信小程序代码上传需要本地代码包、上传密钥和微信 CI 工具。中台负责保存租户配置，代码上传由本地脚本完成。

4. 上传命令等同于微信开发者工具的“上传”吗？

基本等同。tenant:push 在上传前额外做了两件事：

  • 从线上中台同步租户配置。
  • 校验并切换本地小程序构建配置。

但提交审核和发布仍需在微信公众平台人工操作。

5. 哪些改动不需要重新上传小程序？

一般业务内容不需要重新上传，例如：

  • 新增期次
  • 修改课程内容
  • 修改报名价格
  • 调整活动
  • 修改打卡配置

涉及小程序壳的改动通常需要重新上传，例如：

  • appId
  • 品牌名
  • 主色调
  • Logo
  • Tab 图标
  • 分享封面
  • 订阅消息模板键变更
