## 一、支付流程（优先测试，钱相关）

- [ ] **1. 正常支付流程不受影响**  
  操作：小程序正常下单 → 拉起微信支付 → 支付成功 → 确认支付  
  预期：报名状态变为已支付，全流程无报错

- [ ] **2. 支付确认不能重复触发**（C3）  
  操作：支付成功后快速连续点两次确认，或支付后刷新页面再次触发确认  
  预期：报名只激活一次，通知只发一条，无重复记录

- [ ] **3. Mock 支付在生产环境不可用**（C2）  
  操作：确认生产服务器 NODE_ENV=production；调用 `POST /api/v1/payments/initiate` 时 body 传 `paymentMethod: 'mock'`  
  预期：返回 400，不创建报名记录

- [ ] **4. 微信支付回调正常处理**（M4）  
  操作：完成一笔真实微信支付，等待微信服务器回调  
  预期：订单状态正确更新为已支付，报名激活

- [ ] **5. 管理员「标记免费」功能正常**（M3）  
  操作：admin 后台 → 报名管理 → 找一条未支付报名 → 点「标记免费」  
  预期：支付状态变为已支付，页面正常显示
