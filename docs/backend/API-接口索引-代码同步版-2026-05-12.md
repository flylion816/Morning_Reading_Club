> 同步日期：2026-05-12

## 1. 鉴权说明

| 标记 | 含义 |
| --- | --- |
| Public | 无需 token |
| User | `Authorization: Bearer <user_token>`，小程序用户 JWT |
| OptionalUser | 可带用户 JWT，不带也可访问 |
| AdminUser | 用户 JWT 且满足旧 `adminMiddleware` |
| Admin | 管理后台管理员 JWT |
| SuperAdmin | 管理后台管理员 JWT + `requireRole('superadmin')` |

## 3. 已发现的接口一致性问题

- 小程序服务层存在一些未在 `backend/src/app.js` 当前挂载路由中确认的旧接口，例如 `/courses/*`、`/checkins/today`、`/insights/my`、`/insights/public`、`/payments/orders/*`、`/rankings/*`。这些应按实际运行网关、历史兼容或待清理逐项确认。
- `insight.routes.js` 中 `PUT /:insightId` 同时存在用户鉴权和管理员鉴权定义，Express 会按顺序匹配，建议后续拆分管理路径以降低歧义。
- 旧 Postman 文档含真实 token/密码示例，建议替换为占位符，敏感凭据迁移到安全凭据管理。
