# 凡人共读小程序 PRD（代码同步版）

> 同步日期：2026-05-12  
> 同步范围：`miniprogram/`、`backend/src/`、`admin/src/` 当前代码  
> 文档性质：当前产品事实基线。历史 v3/v4/v6 文档保留作背景，但后续需求、设计和测试应优先引用本文件。

## 1. 产品定位

凡人共读是围绕《高效能人士的七个习惯》构建的微信小程序学习营。产品通过期次、每日任务、打卡、互动评论、排行榜、小凡看见和订阅通知，支持用户在固定周期内完成晨读学习、记录实践、查看反馈并参与社群互动。

当前系统由三部分组成：

- 微信小程序：用户学习、报名、支付、打卡、分享、通知和个人主页。
- Node.js 后端：用户认证、课程期次、打卡、评论、小凡看见、支付、通知、管理后台和备份同步。
- Vue 管理后台：期次、课节、用户、报名、支付、打卡、小凡看见、申请、审计和订阅调试管理。

## 2. 用户角色

| 角色 | 来源 | 核心权限 |
| --- | --- | --- |
| 游客 | 未登录小程序用户 | 浏览公开期次、课节概要和部分公开内容；触发受保护操作时进入登录流程 |
| 普通用户 | 微信登录用户，`User.role=user` | 报名、支付、学习、打卡、评论、点赞、查看通知、查看授权范围内的小凡看见 |
| 管理员 | 管理后台 `Admin.role=admin/operator` | 访问管理后台功能，按后端路由和中间件限制执行管理操作 |
| 超级管理员 | 管理后台 `Admin.role=superadmin` | 管理管理员账号、角色、状态和敏感配置 |
| 外部系统 | 明文/受控接口调用方 | 可通过外部接口创建小凡看见或查询活动期次/用户，需由部署环境控制访问范围 |

## 3. 核心业务模块

### 3.1 认证与用户

- 小程序使用 `POST /api/v1/auth/wechat/login` 通过微信 `code` 登录。
- 登录后小程序将 token 和用户信息写入本地存储，并同步到 `app.globalData`。
- 用户资料接口统一使用 MongoDB `_id` 作为主用户 ID。
- 用户头像支持 `avatarUrl` 图片、`avatar` emoji/文本和按用户 ID 生成的稳定背景色。

### 3.2 期次与课节

- 期次（Period）定义学习营周期、起止日期、价格、状态、报名人数和课节数量。
- 课节（Section）挂在期次下，通过 `periodId + day` 唯一约束表示第 N 天任务。
- 小程序首页、晨读营页、课程详情页分别消费期次列表、课节列表和课节详情。
- 管理后台可创建、编辑、复制和删除期次/课节。

### 3.3 报名与支付

- 用户报名生成 Enrollment，状态包括 `active/completed/withdrawn`。
- 支付记录 Payment 关联 Enrollment、User、Period，状态包括 `pending/processing/completed/failed/cancelled`。
- 当前代码同时保留真实微信支付、mock 支付和管理端取消/核销相关接口。
- 报名、支付状态共同决定用户是否能进入学习、打卡和完整互动流程。

### 3.4 打卡与社群互动

- 用户每天对课节提交 Checkin，`userId + sectionId` 唯一，避免同一课节重复打卡。
- 打卡可公开显示在课程详情社区区域，支持点赞和评论。
- 评论模型使用独立 Comment 文档，支持一级评论、嵌套回复、点赞计数和 likes 数组。
- 小程序课程详情支持动态详情模式：从列表、通知、个人主页进入单条打卡详情。

### 3.5 打卡长图分享

当前实现以 `miniprogram/pages/course-detail/course-detail.js` 为准：

- 用户在打卡详情页点击“长图分享”进入模板面板。
- 模板列表来自内置 `POSTER_STYLE_PRESETS`，当前包含青岚、暮紫、晴空、留白。
- 为避免长文和多模板同时生成导致 canvas 超时，系统采用懒生成：
  - 打开面板时只生成首个模板；
  - 其他模板显示轻量预览骨架；
  - 用户点击某模板时才生成该模板真实长图。
- 长文优先完整生成；若完整导出失败，再按高度生成截断版，并提示进入小程序查看完整内容。
- 长图导出对超长图片使用较低倍率和 JPG，降低微信 canvas 内存压力。

### 3.6 小凡看见

- Insight 是对用户打卡/学习状态的反馈内容，支持 `daily/weekly/monthly/insight` 类型。
- 小凡看见可以由后台手动创建、用户手动创建、生成接口创建或外部接口创建。
- 小程序支持查看自己的小凡看见，也支持向他人申请查看权限。
- InsightRequest 记录申请、审批、拒绝、撤回和管理员审批轨迹。
- 详情页支持小凡看见长图分享：
  - HTML/Markdown 内容渲染后解析为 canvas 文本；
  - 支持段落、列表项、引用、标题、普通/加粗混排；
  - 混排文本在样式边界处按宽度换行，避免跑出右边界。

### 3.7 通知与订阅消息

- Notification 存储用户站内通知、阅读状态、归档状态和跳转数据。
- SubscribeMessageGrant 存储订阅模板授权余量和调度状态。
- SubscribeMessageDelivery 记录订阅消息发送结果。
- 小程序通知页、通知设置页、自动补授权工具共同消费这些数据。

### 3.8 排行榜与活动追踪

- Ranking 接口按期次返回排行榜。
- UserActivity 记录用户行为，如打卡、评论、点赞、查看小凡看见等，用于活跃度分析和管理后台统计。

### 3.9 管理后台

管理后台当前页面覆盖：

- Dashboard、Analytics、Users、Periods、Enrollments、Payments
- CheckinsManagement、InsightsManagement、InsightRequestsManagement
- ContentManagement、AccountManagement、AuditLogs、Database、SubscriptionDebug

管理端认证使用独立 Admin 模型和 `adminAuthMiddleware`。部分超级管理员能力使用 `requireRole('superadmin')` 限制。

## 4. 权限边界

| 能力 | 权限要求 |
| --- | --- |
| 微信登录、刷新 token | 无需已有登录 |
| 查看公开期次/课节 | 可选登录或公开 |
| 报名、支付、打卡、评论、点赞 | 小程序用户 JWT |
| 查看自己的小凡看见 | 小程序用户 JWT |
| 查看他人小凡看见 | 小程序用户 JWT + 已授权申请或后台策略 |
| 管理后台登录 | 管理员邮箱/密码 |
| 管理用户、期次、课节、打卡、小凡看见 | 管理员 JWT |
| 管理管理员账号 | 管理员 JWT + `superadmin` |
| 备份/同步管理 | 管理员 JWT |

风险提示：`checkinConfig.routes.js` 当前代码中管理写接口只使用 `adminAuthMiddleware`，如需限制到真实管理员角色，应补 `requireRole('admin', 'superadmin')` 并同步接口文档。

## 5. 非功能需求

- 安全：用户 JWT 和管理员 JWT 分离校验；生产环境 CORS 由环境变量控制；敏感接口需最小权限。
- 性能：列表类接口使用索引；长图分享使用懒生成和导出降级；Redis 可用于缓存和限流。
- 可用性：健康检查路由包括 `/health`、`/status`、`/ready`、`/live`。
- 可维护性：新增功能应同步更新本 PRD、接口索引、数据模型和测试说明。

## 6. 待确认问题

- 小程序服务层仍引用部分后端未挂载或旧命名接口，例如 `/courses/*`、`/insights/my`、`/checkins/today` 等；应逐项确认是否为遗留、代理兼容或待实现。
- API 文档历史版本中包含线上 token 和密码示例，建议改为占位符并移入安全凭据管理流程。
- OpenSpec CLI 已作为 devDependency 接入，可通过 `npm run openspec:list` 查看变更，通过 `npm run openspec:validate` 校验当前 changes。
