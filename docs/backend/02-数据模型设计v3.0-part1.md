## 一、实体概览

### 1.1 核心实体列表

| 序号 | 实体名称 | 英文名称       | 说明                         |
| ---- | -------- | -------------- | ---------------------------- |
| 1    | 用户     | User           | 系统用户基本信息             |
| 2    | 期次     | Period         | 晨读营期次                   |
| 3    | 课节     | Section        | 每天的学习课节               |
| 4    | 打卡     | Checkin        | 用户打卡记录                 |
| 5    | 评论     | Comment        | 对打卡的评论                 |
| 6    | 回复     | Reply          | 对评论的回复                 |
| 7    | 小凡看见 | Insight        | AI生成的反馈                 |
| 8    | 查看申请 | InsightRequest | 查看他人小凡看见的申请       |
| 9    | 报名记录 | Enrollment     | 用户报名期次记录             |
| 10   | 文件     | File           | 上传的文件（图片/视频/音频） |

### 1.2 实体关系图（文字描述）

```
User (用户)
  ├── 1:N → Enrollment (报名记录)
  ├── 1:N → Checkin (打卡)
  ├── 1:N → Comment (评论)
  ├── 1:N → Reply (回复)
  ├── 1:N → Insight (小凡看见)
  └── 1:N → InsightRequest (查看申请)

Period (期次)
  ├── 1:N → Section (课节)
  ├── 1:N → Enrollment (报名记录)
  └── 1:N → Checkin (打卡)

Section (课节)
  ├── N:1 → Period (期次)
  ├── 1:N → Checkin (打卡)
  └── 1:N → Insight (小凡看见)

Checkin (打卡)
  ├── N:1 → User (用户)
  ├── N:1 → Section (课节)
  ├── 1:N → Comment (评论)
  ├── 1:N → File (文件)
  └── 1:1 → Insight (小凡看见)

Comment (评论)
  ├── N:1 → User (用户)
  ├── N:1 → Checkin (打卡)
  └── 1:N → Reply (回复)

Insight (小凡看见)
  ├── N:1 → User (用户)
  ├── N:1 → Checkin (打卡)
  └── 1:N → InsightRequest (查看申请)
```

## 二、详细实体定义

| unionid               | String   | 64   | 否   | null    | 微信UnionID            | oTmHYjg-tElZ9uUkEW7aZr       |
| nickname              | String   | 50   | 是   | -       | 用户昵称               | 阿泰                         |
| avatar                | String   | 10   | 否   | 🦁      | 头像emoji              | 🦁 或 泰                     |
| avatarUrl             | String   | 500  | 否   | null    | 微信头像URL            | https://...                  |
| signature             | String   | 200  | 否   | null    | 个性签名               | 天天开心，觉知当下！         |
| gender                | Enum     | -    | 否   | unknown | 性别                   | male/female/unknown          |
| country               | String   | 50   | 否   | null    | 国家                   | 中国                         |
| province              | String   | 50   | 否   | null    | 省份                   | 广东                         |
| city                  | String   | 50   | 否   | null    | 城市                   | 深圳                         |
| language              | String   | 20   | 否   | zh_CN   | 语言                   | zh_CN                        |
| totalCheckinDays      | Integer  | -    | 否   | 0       | 累计打卡天数           | 23                           |
| currentStreak         | Integer  | -    | 否   | 0       | 当前连续打卡天数       | 5                            |
| maxStreak             | Integer  | -    | 否   | 0       | 最长连续打卡天数       | 15                           |
| totalCompletedPeriods | Integer  | -    | 否   | 0       | 完成的期数             | 3                            |
| totalPoints           | Integer  | -    | 否   | 0       | 总积分                 | 500                          |
| level                 | Integer  | -    | 否   | 1       | 用户等级               | 1                            |
| role                  | Enum     | -    | 否   | user    | 角色                   | user/admin                   |
| status                | Enum     | -    | 否   | active  | 状态                   | active/banned/deleted        |
| lastLoginAt           | DateTime | -    | 否   | null    | 最后登录时间           | 2025-01-13 10:30:00          |
| createdAt             | DateTime | -    | 是   | now()   | 创建时间               | 2025-01-10 08:00:00          |
| updatedAt             | DateTime | -    | 是   | now()   | 更新时间               | 2025-01-13 10:30:00          |

**索引**:

- PRIMARY KEY: id
- UNIQUE INDEX: openid
- INDEX: unionid
- INDEX: nickname
- INDEX: createdAt

**枚举类型**:

```javascript
// 性别
Gender = {
  MALE: 'male',
  FEMALE: 'female',
  UNKNOWN: 'unknown'
};

// 角色
Role = {
  USER: 'user',
  ADMIN: 'admin',
  SUPER_ADMIN: 'super_admin'
};

// 状态
UserStatus = {
  ACTIVE: 'active',
  BANNED: 'banned',
  DELETED: 'deleted'
};
```

**验证规则**:

- nickname: 2-50字符
- signature: 0-200字符
- avatar: 1-10字符（emoji）
- avatarUrl: 有效URL格式

| startDate         | Date     | -    | 是   | -           | 开始日期            | 2025-10-11                    |
| endDate           | Date     | -    | 是   | -           | 结束日期            | 2025-11-13                    |
| totalDays         | Integer  | -    | 否   | 23          | 总天数              | 23                            |

### 2.4 Checkin (打卡)

**表名**: `checkins`

**说明**: 用户每天的打卡记录。

**字段列表**:

| 字段名       | 类型     | 长度 | 必填 | 默认值 | 说明             | 示例                    |
| ------------ | -------- | ---- | ---- | ------ | ---------------- | ----------------------- |
| id           | Integer  | -    | 是   | 自增   | 打卡ID（主键）   | 1                       |
| userId       | Integer  | -    | 是   | -      | 用户ID（外键）   | 101                     |
| periodId     | Integer  | -    | 是   | -      | 期次ID（外键）   | 8                       |
| sectionId    | Integer  | -    | 是   | -      | 课节ID（外键）   | 802                     |
| content      | Text     | -    | 是   | -      | 打卡内容         | 今天学习了品德成功论... |
| visibility   | Enum     | -    | 否   | all    | 可见范围         | all/admin_only          |
| likeCount    | Integer  | -    | 否   | 0      | 点赞数           | 5                       |
| commentCount | Integer  | -    | 否   | 0      | 评论数           | 3                       |
| isLiked      | Boolean  | -    | 否   | false  | 当前用户是否点赞 | true                    |
| status       | Enum     | -    | 否   | normal | 状态             | normal/deleted/hidden   |
| createdAt    | DateTime | -    | 是   | now()  | 创建时间         | 2025-10-11 07:30:00     |
| updatedAt    | DateTime | -    | 是   | now()  | 更新时间         | 2025-10-11 07:30:00     |

**索引**:

- PRIMARY KEY: id
- UNIQUE INDEX: userId, sectionId (一个用户每个课节只能打卡一次)
- INDEX: periodId, createdAt DESC (期次打卡列表)
- INDEX: sectionId, createdAt DESC (课节打卡列表)
- INDEX: userId, createdAt DESC (用户打卡列表)

**外键约束**:

- FOREIGN KEY (userId) REFERENCES users(id) ON DELETE CASCADE
- FOREIGN KEY (periodId) REFERENCES periods(id) ON DELETE CASCADE
- FOREIGN KEY (sectionId) REFERENCES sections(id) ON DELETE CASCADE

**枚举类型**:

```javascript
CheckinVisibility = {
  ALL: 'all', // 所有人可见
  ADMIN_ONLY: 'admin_only' // 仅管理员可见
};

CheckinStatus = {
  NORMAL: 'normal', // 正常
  DELETED: 'deleted', // 已删除
  HIDDEN: 'hidden' // 已隐藏
};
```

**关联数据**:

- images: String[] - 图片URL数组
- videos: String[] - 视频URL数组
- voices: String[] - 音频URL数组

**计算字段**:

- `userName`: 用户昵称
- `avatarText`: 用户头像文字
- `avatarColor`: 用户头像颜色
- `sectionTitle`: 课节标题
- `sectionDay`: 课节天数
- `createTimeFormatted`: 格式化的创建时间（如"1小时前"）

## 三、数据关系详解

### 3.1 一对多关系 (1:N)

**User → Enrollment**

- 一个用户可以报名多个期次
- 通过 `enrollments.userId` 关联

**User → Checkin**

- 一个用户可以有多个打卡记录
- 通过 `checkins.userId` 关联

**User → Comment**

- 一个用户可以发表多条评论
- 通过 `comments.userId` 关联

**User → Reply**

- 一个用户可以发表多条回复
- 通过 `replies.userId` 关联

**User → Insight**

- 一个用户可以有多个小凡看见
- 通过 `insights.userId` 关联

**Period → Section**

- 一个期次包含多个课节
- 通过 `sections.periodId` 关联

**Period → Enrollment**

- 一个期次可以有多个报名记录
- 通过 `enrollments.periodId` 关联

**Section → Checkin**

- 一个课节可以有多个打卡记录
- 通过 `checkins.sectionId` 关联

**Checkin → Comment**

- 一个打卡可以有多条评论
- 通过 `comments.checkinId` 关联

**Checkin → File**

- 一个打卡可以有多个文件（图片/视频/音频）
- 通过 `files.relatedId` (relatedType='checkin') 关联

**Comment → Reply**

- 一条评论可以有多条回复
- 通过 `replies.commentId` 关联

**Insight → InsightRequest**

- 一个小凡看见可以有多个查看申请
- 通过 `insight_requests.insightId` 关联

### 3.2 一对一关系 (1:1)

**Checkin ↔ Insight**

- 一个打卡对应一个小凡看见
- 通过 `insights.checkinId` (UNIQUE) 关联

### 3.3 多对多关系

**用户点赞打卡** (通过中间表 `checkin_likes`)

- 字段：id, userId, checkinId, createdAt
- 唯一索引：(userId, checkinId)

**用户点赞评论** (通过中间表 `comment_likes`)

- 字段：id, userId, commentId, createdAt
- 唯一索引：(userId, commentId)

**用户点赞回复** (通过中间表 `reply_likes`)

- 字段：id, userId, replyId, createdAt
- 唯一索引：(userId, replyId)

## 四、业务规则和约束

### 4.1 唯一性约束

1. **用户维度**
   - `users.openid`: 微信OpenID全局唯一
   - `enrollments(userId, periodId)`: 每个用户每个期次只能报名一次
   - `checkins(userId, sectionId)`: 每个用户每个课节只能打卡一次
   - `insight_requests(insightId, fromUserId)`: 同一个小凡看见每个用户只能申请一次

2. **数据维度**
   - `insights.checkinId`: 每个打卡只有一个小凡看见

### 4.2 级联删除规则

1. **用户删除**
   - 删除用户时，级联删除该用户的所有：报名记录、打卡、评论、回复、小凡看见、文件

2. **期次删除**
   - 删除期次时，级联删除该期次的所有：课节、报名记录、打卡

3. **课节删除**
   - 删除课节时，级联删除该课节的所有打卡

4. **打卡删除**
   - 删除打卡时，级联删除该打卡的所有：评论、文件、小凡看见

5. **评论删除**
   - 删除评论时，级联删除该评论的所有回复

6. **小凡看见删除**
   - 删除小凡看见时，级联删除所有查看申请

### 4.3 软删除字段

以下实体支持软删除（通过status字段标记为deleted）：

- User
- Checkin
- Comment
- Reply
- Insight
- File

### 4.4 计数器字段同步规则

1. **打卡点赞数** (`checkins.likeCount`)
   - 新增点赞：+1
   - 取消点赞：-1

2. **打卡评论数** (`checkins.commentCount`)
   - 新增评论：+1
   - 删除评论：-1

3. **评论回复数** (`comments.replyCount`)
   - 新增回复：+1
   - 删除回复：-1

4. **小凡看见查看次数** (`insights.viewCount`)
   - 每次查看：+1

5. **小凡看见申请次数** (`insights.requestCount`)
   - 新增申请：+1

6. **用户打卡统计**
   - `users.totalCheckinDays`: 累计打卡天数（新增打卡时+1）
   - `users.currentStreak`: 当前连续打卡天数（连续打卡时+1，中断时重置为1）
   - `users.maxStreak`: 最长连续打卡天数（当currentStreak > maxStreak时更新）

7. **期次报名人数** (`periods.currentEnrollment`)
   - 新增报名：+1
   - 取消报名：-1

## 七、索引设计原则

### 7.1 主键索引

- 所有表的 `id` 字段作为主键
- 自动创建聚簇索引（MySQL InnoDB）

### 7.2 唯一索引

用于保证业务唯一性：

- `users.openid`
- `enrollments(userId, periodId)`
- `checkins(userId, sectionId)`
- `insights.checkinId`
- `insight_requests(insightId, fromUserId)`

### 7.3 外键索引

用于关联查询和级联操作：

- 所有外键字段（userId, periodId, sectionId等）

### 7.4 复合索引

用于优化列表查询：

- `(periodId, createdAt DESC)`: 期次相关的时间序列查询
- `(userId, createdAt DESC)`: 用户相关的时间序列查询
- `(sectionId, createdAt DESC)`: 课节相关的时间序列查询
- `(isPublished, sortOrder)`: 发布内容的排序查询

### 7.5 索引优化建议

1. **选择性高的字段优先建索引**
2. **常用于WHERE、ORDER BY的字段建索引**
3. **外键字段必须建索引**
4. **避免过多索引影响写入性能**
5. **定期分析索引使用情况，删除无用索引**

## 九、性能优化建议

### 9.1 数据库优化

1. **合理使用索引**
   - 为常用查询字段建立索引
   - 避免过多索引

2. **读写分离**
   - 配置主从复制
   - 读操作走从库

3. **分库分表**
   - 按期次分表（如果数据量大）
   - 按时间分表（如打卡记录按年月分表）

### 9.2 缓存策略

1. **热点数据缓存**
   - 期次列表（15分钟）
   - 课节内容（30分钟）
   - 用户信息（5分钟）

2. **计数器缓存**
   - 点赞数、评论数使用Redis计数器
   - 定期同步到数据库

3. **查询结果缓存**
   - 排行榜（1分钟）
   - 打卡列表（30秒）

### 9.3 查询优化

1. **使用分页**
   - 所有列表接口必须分页
   - 默认每页20条

2. **避免N+1查询**
   - 使用JOIN或批量查询
   - 减少数据库往返次数

3. **只查询需要的字段**
   - 避免SELECT \*
   - 使用字段投影

## 十、数据安全和隐私

### 10.1 敏感数据处理

1. **微信OpenID**
   - 不对外暴露
   - 仅用于内部关联

2. **用户地理位置**
   - country, province, city
   - 仅管理员可见完整信息

3. **小凡看见内容**
   - 默认私有
   - 需要申请才能查看

### 10.2 数据访问控制

1. **用户只能访问自己的数据**
   - 打卡记录
   - 小凡看见
   - 报名记录

2. **管理员权限**
   - 可以查看所有数据
   - 可以管理所有内容

3. **隐私设置**
   - 打卡可见性：all/admin_only
   - 小凡看见可见性：private/friends/public

### 10.3 数据备份

1. **定期备份**
   - 每天全量备份
   - 每小时增量备份

2. **备份保留**
   - 全量备份保留30天
   - 增量备份保留7天

3. **灾难恢复**
   - RPO（恢复点目标）：1小时
   - RTO（恢复时间目标）：4小时
