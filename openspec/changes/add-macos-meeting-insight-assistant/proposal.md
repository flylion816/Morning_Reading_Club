# 变更：新增 macOS 会议洞察助手方案

## Why
用户希望在下一期晨读营中作为观察者，能快速承接多位学员的线上分享，抓住每个人发言背后的本质点，并结合当期七个习惯主题和自己的破产重建经历，给出真诚、克制、有分量的现场回应。原始输入可以来自腾讯会议转写、截图 OCR、手动速记或未来音频转写；真正优先要解决的是多人发言的采集、单人洞察、现场回应和全场合并工作流。

## What Changes
- 新增一个独立 macOS “晨读观察台”方案，优先验证多人发言卡片、剪贴板粘贴、单人洞察、现场回应和全场总结的工作流。
- 方案采用观察者工作台、发言人卡片、主题锚点、个人故事库、AI 结构化分析、全场合并和会后沉淀七层架构。
- 语音采集和实时 ASR 保留为后续输入源，不作为第一版 MVP 的核心路径。
- 方案明确插件使用分工：`Product Design` 用于产品流程和界面原型，`GitHub` 用于后续实现和版本管理，`Browser` 用于本地界面测试，`Documents` 用于正式方案/纪要文档产出，`Hugging Face` 仅用于离线 ASR 模型评估。
- 方案不把会议助手直接接入当前晨读营小程序、Node 后端、MongoDB 或管理后台；后续实现建议新建独立仓库或独立 workspace 子目录。

## Impact
- 影响规格：`meeting-insight-assistant`
- 影响代码：P0 阶段建议在 `meeting-assistant/` 中实现独立观察者工作台，不修改 `miniprogram/`、`backend/`、`admin/`。
- 外部依赖：OpenAI API 或其他大模型 API；后续可选 macOS 系统音频权限、麦克风权限、腾讯会议转写 webhook、腾讯云实时语音识别 WebSocket、可选本地 ASR 模型。
- 数据安全：P0 不读取或修改晨读营生产数据库，不执行任何数据库初始化、重置或迁移脚本。
