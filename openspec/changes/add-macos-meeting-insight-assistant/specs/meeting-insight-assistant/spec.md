## ADDED Requirements

### Requirement: 多发言人观察者工作台
系统 MUST 提供一个面向晨读营观察者的多人工作台，支持在同一场会话中管理多位发言人的内容、状态、单人洞察和现场回应。

#### Scenario: 创建多位发言人卡片
- **GIVEN** 用户已创建一场观察者会话
- **WHEN** 用户新增四位发言人
- **THEN** 系统展示四张发言人卡片
- **AND** 每张卡片显示姓名、状态和是否已生成回应。

#### Scenario: 删除发言人卡片
- **GIVEN** 会话中存在多位发言人
- **WHEN** 用户删除某位发言人并确认
- **THEN** 系统删除该发言人及其内容片段、单人洞察和实际回应
- **AND** 系统至少保留一位发言人。

#### Scenario: 切换当前发言人
- **GIVEN** 会话中已有多位发言人
- **WHEN** 用户点击某位发言人卡片
- **THEN** 系统将其设为当前发言人
- **AND** 中间内容区展示该发言人的内容片段和单人洞察。

#### Scenario: 收起书友分享资料面板
- **GIVEN** 用户正在查看发言人列表、内容片段和现场回应
- **WHEN** 用户点击书友分享资料面板右上角的收起按钮
- **THEN** 系统隐藏发言人列表、内容片段、观察书友分享结果和现场回应
- **AND** 面板标题区仍显示已采集人数、已观察人数和展开按钮。

### Requirement: 多段内容粘贴
系统 MUST 允许用户把多段来自不同来源的文字粘贴到同一位或不同发言人的卡片中，并保留每段内容的来源和创建时间。

#### Scenario: 粘贴到当前发言人
- **GIVEN** 用户已选中某位发言人
- **AND** 剪贴板中有腾讯会议转写或 OCR 后文本
- **WHEN** 用户点击“粘贴到当前发言人”
- **THEN** 系统将剪贴板文本新增为该发言人的一个内容片段
- **AND** 该发言人状态变为 `captured`。

#### Scenario: 同一发言人追加多段内容
- **GIVEN** 某发言人已有一段内容片段
- **WHEN** 用户再次向该发言人粘贴新内容
- **THEN** 系统保留两段内容
- **AND** 单人分析时同时使用该发言人的全部内容片段。

#### Scenario: 上传截图图片到当前发言人
- **GIVEN** 用户已选中某位发言人
- **WHEN** 用户上传一张会议字幕、聊天或 OCR 前截图
- **THEN** 系统将图片保存为该发言人的内容片段
- **AND** 内容片段列表展示该图片的缩略图和文件名。

#### Scenario: 直接粘贴剪贴板截图
- **GIVEN** 用户已用系统截图工具把截图放入剪贴板
- **AND** 用户已选中某位发言人
- **WHEN** 用户在内容输入框中粘贴或点击“粘贴截图”
- **THEN** 系统将剪贴板截图加入当前发言人的内容片段
- **AND** 系统自动执行本机 OCR 并保存识别文字。

### Requirement: 当天课程锚点
系统 MUST 允许用户在会话顶部选择 23 天课程中的当天课程，并在所有单人分析和全场合并中使用该课程作为分析锚点。

#### Scenario: 选择当天课程
- **GIVEN** 用户正在创建观察者会话
- **WHEN** 用户选择某一天的课程题目
- **THEN** 系统保存该课程
- **AND** 后续分析围绕当天课程内容生成。

#### Scenario: 查看当天课程内容
- **GIVEN** 用户已选择某一天课程
- **WHEN** 用户点击“课程内容”
- **THEN** 系统展示当天课程正文、课程核心和观察问题。

#### Scenario: 按课程天数恢复内容
- **GIVEN** 用户在某一天课程下已采集发言人内容并生成分析
- **WHEN** 用户切换到其他课程后再切回该课程
- **THEN** 系统恢复该课程下保存的发言人、内容片段、单人分析和全场观察。

#### Scenario: 清空当天内容
- **GIVEN** 当前课程下已有发言人内容、单人分析或全场观察
- **WHEN** 用户点击“清空当天”并确认
- **THEN** 系统仅清空当前课程下的会话内容
- **AND** 其他课程天数下保存的内容不受影响。

### Requirement: 单人洞察和现场回应
系统 MUST 为单位发言人生成结构化洞察和现场回应，包括最值得抓的点、背后本质、课程连接、回应稿、追问、金句和不要说的话。

#### Scenario: 生成单人洞察
- **GIVEN** 某发言人已有至少一段内容片段
- **AND** 会话已选择本期主题
- **WHEN** 用户点击“观察书友分享”
- **THEN** 系统生成最值得抓的点、背后本质、主题连接、卡点分类和需要被看见的地方
- **AND** 系统保存分析结果到该发言人卡片。

#### Scenario: 自动替换默认发言人名称
- **GIVEN** 某发言人的名称仍为“发言人 A”一类默认名称
- **AND** 粘贴或 OCR 内容中包含明确发言人姓名
- **WHEN** 系统生成单人洞察
- **THEN** 系统将该卡片名称更新为“发言人 姓名”。

#### Scenario: 基于上传图片生成洞察
- **GIVEN** 某发言人已有一张会议截图图片片段
- **AND** 当前模型服务支持图片输入
- **WHEN** 用户点击“观察书友分享”
- **THEN** 系统将图片作为多模态输入发送给模型
- **AND** 模型先识别图片中的可见文字，再生成单人洞察。

#### Scenario: 生成现场回应
- **GIVEN** 某发言人已有单人洞察
- **WHEN** 用户点击“生成现场回应”
- **THEN** 系统生成 1 分钟回应、有力量的追问和金句
- **AND** 系统同时生成“不要说的话”。

### Requirement: AI 模型服务设置
系统 MUST 支持用户在本机设置 AI 模型服务，并至少支持 OpenAI Responses、小米 MiMo 和自定义 OpenAI 兼容接口。

#### Scenario: 使用小米 MiMo 生成洞察
- **GIVEN** 用户在设置中选择“小米 MiMo”
- **AND** 用户填写小米 API Key、Base URL 和模型
- **WHEN** 用户点击“观察书友分享”
- **THEN** 系统通过 OpenAI Chat Completions 兼容接口请求模型
- **AND** 系统将模型返回的 JSON 保存为该发言人的洞察。

#### Scenario: 没有 API Key 时使用本地演练稿
- **GIVEN** 用户没有填写任何 AI API Key
- **WHEN** 用户点击“观察书友分享”或“输出全场观察”
- **THEN** 系统生成本地演练稿
- **AND** 系统提示用户设置 API Key 后可获得真实 AI 分析。

### Requirement: 个人故事库匹配
系统 MUST 支持观察者个人故事库，并在单人洞察中推荐适合关联的个人经历、适合说到的程度和不应展开的细节。

#### Scenario: 推荐观察者经历
- **GIVEN** 个人故事库包含破产、债务、银行起诉、限高和重新站起来等经历
- **WHEN** 系统为某位发言人生成单人洞察
- **THEN** 系统推荐一个最适合关联的经历
- **AND** 系统说明为什么适合、应该说到什么程度、哪些细节不要展开。

#### Scenario: 维护观察者知识库
- **GIVEN** 观察者拥有过去晨读营发言、破产重建经历和可复用回应片段
- **WHEN** 用户打开“我的知识库”
- **THEN** 系统允许用户新增、编辑、删除知识条目
- **AND** 每个条目可记录类型、标签、适合场景、摘要、可引用原话、讲述边界和不应展开的细节。

#### Scenario: 从长文档提炼知识条目
- **GIVEN** 用户有一篇过去发言文档或大段文字
- **WHEN** 用户粘贴文字或上传 txt、md、docx 并点击“AI 提炼成知识条目”
- **THEN** 系统将原文提炼为一个或多个结构化知识条目
- **AND** 系统保留原话片段、可复用句子和讲述边界，避免直接把整篇原文塞入分析 prompt。

#### Scenario: 分析时匹配知识库
- **GIVEN** 知识库中已有观察者过去发言和破产重建素材
- **WHEN** 用户点击“观察书友分享”或“输出全场观察”
- **THEN** 系统按当前课程和发言内容匹配相关知识条目
- **AND** 系统将相关条目的摘要、适合场景、可引用原话、讲述边界和不应展开的细节提供给 AI。

### Requirement: 全场统一洞察
系统 MUST 支持将多位发言人的内容和单人分析合并成多版本全场统一洞察，并允许在未全部分析时生成草稿、比较模板、追加优化版和保存最终发言。

#### Scenario: 输出三套全场观察
- **GIVEN** 会话中已有多位发言人的单人分析
- **WHEN** 用户点击“输出全场观察”
- **THEN** 系统先输出自动推荐或用户指定的模板版本
- **AND** 同时输出剩余两套模板版本供用户对照选择
- **AND** 每个版本都包含结构化分享板块、金句、一句话总结和收束句
- **AND** 每个版本应围绕当天课程、多位发言人的共同课题和观察者自身破产重建经历生成可直接口播内容。

#### Scenario: 再次输出替换三套模板
- **GIVEN** 用户已输出过三套全场观察版本
- **AND** 用户又新增或修改了上方发言人内容
- **WHEN** 用户再次点击“再次输出全场观察”或后续次数按钮
- **THEN** 系统重新生成三套模板版本
- **AND** 新三套模板版本 MUST 替换旧三套模板版本，不得每次追加三套新版本。

#### Scenario: 按优化要求新增版本
- **GIVEN** 用户已生成至少一版全场观察
- **WHEN** 用户在“我的优化要求”中写明基于哪一版调整并点击“新增优化版”
- **THEN** 系统基于已有版本、用户要求、课程内容、发言人内容和知识素材生成新的优化版本
- **AND** 系统不得覆盖已存在的三套模板版本或历史优化版本。

#### Scenario: 管理单个总结版本
- **GIVEN** 全场统一洞察中已有多个版本
- **WHEN** 用户点击某个版本右上角的收起、展开或删除
- **THEN** 系统仅改变或删除该版本
- **AND** 其他版本、发言人内容和知识库不受影响。

#### Scenario: 保存本次最后的发言
- **GIVEN** 全场统一洞察中已有一个或多个总结版本
- **WHEN** 用户把版本内容放入底部最终发言编辑区并保存
- **THEN** 系统把编辑后的富文本内容保存为当前课程的本次最后发言
- **AND** 整版插入时系统 MUST 将标题、正文、列表、金句和收束句拆成可读的富文本结构
- **AND** 导出 Markdown 复盘时包含该最终发言。

#### Scenario: 复制或插入单个总结板块
- **GIVEN** 全场统一洞察中的某个版本已展开并展示多个结构化板块
- **WHEN** 用户点击某个板块右上角的复制或插入正文
- **THEN** 系统复制该板块标题和完整正文，或把该板块标题和完整正文插入到底部最终发言编辑区末尾
- **AND** 插入正文时如果最终发言已有内容，系统 MUST 先追加一个空行再插入该板块内容
- **AND** 如果最终发言为空，系统 MUST 直接插入该板块内容。

#### Scenario: 未分析发言人提示
- **GIVEN** 会话中存在尚未分析的发言人
- **WHEN** 用户点击“输出全场观察”
- **THEN** 系统提示哪些发言人尚未分析
- **AND** 用户仍可选择生成草稿。

### Requirement: macOS 会议音频采集
系统 MUST 在 macOS 上支持捕获在线会议的系统输出音频，并支持同时捕获本机麦克风音频，且两路音频 MUST 以不同来源保存。

#### Scenario: 捕获腾讯会议系统音频
- **GIVEN** 用户已在 Mac 上加入腾讯会议
- **AND** 用户已授予应用所需音频相关权限
- **WHEN** 用户在会议洞察助手中点击开始监听
- **THEN** 系统捕获腾讯会议输出音频
- **AND** 系统将该音频标记为 `system_audio` 来源。

#### Scenario: 捕获本机麦克风音频
- **GIVEN** 用户已授予麦克风权限
- **WHEN** 用户在会议中发言
- **THEN** 系统捕获麦克风音频
- **AND** 系统将该音频标记为 `microphone` 来源。

#### Scenario: 停止监听后释放采集
- **GIVEN** 系统正在捕获会议音频和麦克风音频
- **WHEN** 用户点击停止监听
- **THEN** 系统停止采集音频
- **AND** 系统不得继续写入新的音频或转写片段。

### Requirement: 实时语音转写
系统 MUST 将捕获到的音频实时发送到语音识别服务，并生成带时间戳、来源、最终状态和文本内容的转写片段。

#### Scenario: 系统音频生成实时转写
- **GIVEN** 系统正在捕获会议系统音频
- **WHEN** 会议中远端参会者发言
- **THEN** 系统展示实时转写文本
- **AND** 生成的片段包含 `sourceId`、`startMs`、`endMs`、`text` 和 `isFinal`。

#### Scenario: 麦克风生成实时转写
- **GIVEN** 系统正在捕获麦克风音频
- **WHEN** 用户本人发言
- **THEN** 系统展示实时转写文本
- **AND** 生成的片段 speaker hint 优先标记为 `self`。

#### Scenario: 转写服务中断
- **GIVEN** 系统正在发送音频到 ASR 服务
- **WHEN** ASR WebSocket 连接中断
- **THEN** 系统提示连接异常
- **AND** 系统尝试按配置重连
- **AND** 系统不得丢失已完成的最终转写片段。

### Requirement: 时间轴分段与选段分析
系统 MUST 将实时转写组织成可选择的时间轴段落，并允许用户对一个或多个段落执行洞察分析。

#### Scenario: 用户选中一个段落做洞察
- **GIVEN** 实时转写时间轴已有多个段落
- **WHEN** 用户选中一个段落并选择“行动项”
- **THEN** 系统基于该段落和必要上下文生成行动项分析
- **AND** 系统保存该分析结果。

#### Scenario: 用户选中多个段落做总结
- **GIVEN** 实时转写时间轴已有多个连续段落
- **WHEN** 用户选择多个段落并选择“总结”
- **THEN** 系统生成覆盖所选段落的摘要
- **AND** 分析结果记录关联的 segment ids。

### Requirement: 多端内容合并
系统 MUST 支持将实时转写、会后转写、手动笔记和额外录音转写合并到同一个会议时间轴，并对重复内容进行基础去重。

#### Scenario: 合并会后转写
- **GIVEN** 某会议已有实时转写时间轴
- **WHEN** 用户导入同一会议的会后转写文本
- **THEN** 系统将会后转写作为新的内容来源
- **AND** 系统按时间或语义相似度与已有片段对齐。

#### Scenario: 去除重复片段
- **GIVEN** 实时转写和会后转写存在相同或高度相似的内容
- **WHEN** 系统执行合并
- **THEN** 系统保留置信度或完整度更高的片段
- **AND** 系统将被合并片段标记为补充来源而不是直接丢弃。

### Requirement: 会议结果导出
系统 MUST 支持把会议时间轴、洞察结果和行动项导出为用户可复制或保存的会议输出。

#### Scenario: 导出 Markdown 会议纪要
- **GIVEN** 会议已有转写片段和洞察结果
- **WHEN** 用户选择导出 Markdown
- **THEN** 系统生成包含会议摘要、关键决策、行动项、风险和原文引用的 Markdown 内容。

#### Scenario: 导出日报素材
- **GIVEN** 用户已对会议片段生成洞察
- **WHEN** 用户选择导出日报素材
- **THEN** 系统输出适合放入日报的事实、判断、行动项和待跟进事项。

### Requirement: 隐私与本地数据控制
系统 MUST 明确展示音频采集状态，并允许用户删除本地会议数据，避免敏感会议内容在本机长期残留。

#### Scenario: 展示采集状态
- **GIVEN** 系统正在监听会议音频
- **WHEN** 用户查看应用主界面
- **THEN** 界面明确显示当前正在采集的来源
- **AND** 提供停止监听入口。

#### Scenario: 删除会议数据
- **GIVEN** 本地已保存某次会议的转写和洞察
- **WHEN** 用户选择删除该会议
- **THEN** 系统删除该会议的本地转写、洞察和来源记录
- **AND** 删除后不得在会议列表中继续展示该会议。
