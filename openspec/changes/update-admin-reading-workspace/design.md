## Context
登录页以摊开的书为主要视觉隐喻。管理后台需要延续这种气质，但运营页面信息密度高，不能把每个页面直接模拟成装饰性书本，也不能为不同租户复制多套 CSS。

## Goals / Non-Goals
- Goals: 统一登录前后气质；提升页面层级与扫描效率；保留租户品牌色；以共享样式覆盖全部后台页面。
- Non-Goals: 不修改业务流程、接口、权限、数据库或路由；不重写各业务页面；不引入新的 UI 框架。

## Decisions
- Decision: 采用“书房工作台”而不是逐页“摊开书本”。侧栏承担书脊/封面角色，页头承担章节眉题角色，内容区使用暖白纸张和细线秩序。
- Decision: 中性色固定为纸张、墨色和边框；租户品牌色只派生操作色、选中态、浅底和阴影，确保不同色值下仍有一致质感与可读性。
- Decision: 通过共享布局和全局 Element Plus 覆盖实现大部分统一，仅清理会冲突的局部硬编码颜色。
- Decision: 卡片圆角保持 8px 以内，页面内容不使用嵌套装饰卡片。

## Alternatives Considered
- 整页模拟摊开的书：视觉最强，但会降低后台信息密度并制造大量嵌套容器。
- 仅替换颜色：风险最低，但无法解决登录前后的结构与层级断裂。

## Risks / Trade-offs
- 深色或过亮租户色可能影响对比度 -> 继续使用亮度检测和混色派生，不直接把原色用于大面积背景。
- 全局样式可能影响复杂页面 -> 先用布局契约测试和全量 Vitest 验证，再逐页截图检查核心路由。
- 窄屏侧栏信息过多 -> 保留现有功能分组，并在窄屏切换为紧凑导航布局。

## Migration Plan
仅前端样式与布局迁移。构建产物可通过上一提交重新部署回滚，无数据迁移。
