# 变更：增加“我发起的请求看见”状态展示

## 为什么
用户目前只能在“请求看见”里看到自己收到的申请，无法方便地查看自己发起的申请处于等待、同意还是拒绝状态。发起申请后缺少反馈，会让用户不确定对方是否已经处理。

## 改动内容
- 在小程序“请求看见”体验中增加“我发起的”视图。
- 每条发起记录展示对方用户、申请的小凡看见上下文、申请时间和当前状态。
- 保持收到请求和发起请求的视觉一致性，同时区分操作状态：收到的待处理请求可同意/拒绝；我发起的请求只展示状态。
- 优化请求卡片视觉：分段切换、清晰状态标签、精简元信息和两个方向各自的空状态。

## 影响范围
- 受影响规格：`insight-requests`
- 受影响代码：`miniprogram/pages/profile/*`、`miniprogram/pages/insight-requests/*`、`miniprogram/services/insight.service.js`
- 后端影响：复用现有 `GET /api/v1/insights/requests/sent` 接口；不做数据结构迁移，不重置数据库。
