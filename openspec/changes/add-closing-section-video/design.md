## Context
结营词本质上仍是一个 `Section` 课节，当前管理后台通过 `ContentManagementView.vue` 编辑课节字段，小程序课程详情页根据 `course` 字段动态展示“静一静、读一读、看一看、播一播”等模块。已有上传能力使用租户目录保存文件并返回 URL，课节模型里已有播客字段，但没有可播放的视频字段，也没有可供分享卡片使用的视频封面字段。

## Goals / Non-Goals
- 目标：管理员可为结营词课节上传一个视频；小程序课节详情可播放该视频；视频可通过微信小程序分享入口分享；结营词普通分享优先使用视频首帧封面。
- 目标：上传文件和课节字段必须按租户隔离，普通用户只能读取已发布课节中的视频信息。
- 非目标：本次不做视频转码、不做 CDN 迁移、不做视频水印、不做独立的视频内容库。
- 非目标：不强制只能最后一课使用该字段；后台文案提示用于结营词，数据模型保留为课节级可选视频，降低运营误配置修复成本。

## Decisions
- 决策：在 `Section` 上增加 `closingVideo` 子对象，而不是新增集合。
  原因：结营视频只属于一个课节，跟课节发布、租户隔离、课程详情读取生命周期一致。
- 决策：`closingVideo` 至少保存 `url` 和 `coverUrl`，并补充 `originalName`、`fileName`、`mimeType`、`size`、`duration`、`uploadedAt`。
  原因：小程序播放使用 `url`，分享封面使用 `coverUrl`；其他元数据用于后台展示和后续排查。
- 决策：首帧封面由管理后台在浏览器端读取本地视频第一帧生成图片后上传，失败时允许管理员仅保存视频，但小程序分享封面回退到默认课程封面。
  原因：微信分享 `imageUrl` 需要图片地址；后端目前没有稳定的视频解码依赖，前端截帧可以避免引入 ffmpeg 类服务端依赖。
- 决策：上传接口限制视频 MIME/扩展名和文件大小，复用租户上传目录，管理后台必须带管理员鉴权和租户上下文。
  原因：视频文件大且风险高，必须避免普通公开接口写入任意文件。
- 决策：小程序“分享结营视频”和“分享结营词”共用课程详情页路径，视频分享带 `anchor=closingVideo` 参数，进入后自动滚动到视频板块。
  原因：保持微信小程序分享为页面分享，不引入外部下载或直链暴露。

## Data Shape
```js
closingVideo: {
  url: String,
  coverUrl: String,
  originalName: String,
  fileName: String,
  mimeType: String,
  size: Number,
  duration: Number,
  uploadedAt: Date
}
```

## Interaction Details
- 后台课节编辑弹窗中，“结营视频”放在扩展信息区域，靠近“凡人播客”，支持选择视频、显示文件名/大小/时长、预览、删除。
- 后台上传视频后尝试自动截取首帧并上传为封面图；如果截帧失败，给出明确提示，但不阻塞视频保存。
- 小程序课程详情在“读一读/看一看”等内容之后展示“结营视频”板块，使用 `<video>` 播放，`poster` 使用 `closingVideo.coverUrl`。
- 视频板块内提供“分享好友”按钮，触发 `open-type="share"`；分享标题使用 `结营视频｜{课程标题}`，路径携带 `anchor=closingVideo`。
- 普通课程分享如果当前课节存在 `closingVideo.coverUrl`，优先使用该图片作为分享卡片封面。

## Risks / Trade-offs
- 视频文件较大可能导致上传超时。缓解方式：设置明确大小限制和上传进度提示，后台保存前不自动发布额外状态。
- 浏览器首帧截取可能受视频编码影响失败。缓解方式：失败时允许保存视频，分享封面回退默认图片；后续可增加手动上传封面图。
- 小程序 `<video>` 在不同基础库下表现略有差异。缓解方式：使用原生 video 组件、稳定尺寸和 poster，不自定义复杂播放器。
- 已发布课节增加视频字段后会立即对小程序可见。缓解方式：后台删除/清空字段即可回滚，不需要数据迁移。

## Migration Plan
- 新字段为可选字段，历史课节默认没有 `closingVideo`，无需数据迁移。
- 部署后先在后台给测试期次结营词上传视频验证，再给正式结营词上传。
- 回滚时清空对应课节的 `closingVideo` 字段即可隐藏小程序视频板块。

## Open Questions
- 视频大小上限建议先按 200MB，与现有播客上传上限保持一致；如果实际结营视频更大，需要单独确认存储和小程序播放体验。
